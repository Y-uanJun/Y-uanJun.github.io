Geolocation API

也许很多人都遇到过类似的经历，在手机上打开网页时，会弹出一个选择框，提示是否允许网站获得您的当前位置，这个就是常见的地理位置功能。
在HTML 5中，为window.navigator对象新增了geolocation属性，可以使用Geolocation API对其进行访问。

方法使用

window.navigator对象的geolocation属性有以下三个方法getCurrentPosition()、watchposition()、clearWatch()。

getCurrentPosition(onSuccess,onError,options)

用来获取用户当前地理位置。
第一个参数onSuccess为获取当前地理位置成功时的回调函数。
navigator.geolocation.getCurrentPosition(position){ //获取成功时的回调函数 }

position对象

在成功获取地理位置信息后。可以通过访问position对象的属性来获得这些地理信息。

第二个参数onError为获取地理位置失败时执行的回调函数。如果用户拒绝共享也会造成错误发生。
该回调函数使用error对象作为参数，该对象有两个属性：
a：code属性

code 属性为以下三个值的其中一个：
用户拒绝位置服务（属性值为1）
获取不到位置信息（属性值为2）
获取信息超时错误（属性值为3）

b：message属性

message属性为一个字符串，该字符串包含了错误信息，有些浏览器不支持这一属性，比如Firefox 3.6以上

navigator.geolocation.getCurrentPosition(function(position){ var coords = position.coords; showMap(coords.latitude,coords.longitude,coords.accuracy); }, function(error){ var errorTypes = { 1:"位置服务被拒绝", 2:"获取不到位置信息", 3:"获取信息超时" }; alert(errorTypes [error.code]+":获取信息失败") }; );
第三个参数是一些可选属性的列表。
timeout：对地理位置信息的获取操作做出一个超时限制，超过时间返回错误。
maximumAge：对地理信息进行缓存的有效时间

多数情况下这些属性设为默认值，有设备自身来调整

watchPosition()

持续监视当前地理位置的信息

clearWatch()

停止对当前地理位置信息的监视
watchId=watchCurrentPosition(onSuccess,onError,options);
//持续监视地理位置信息，返回一个数字
clearWatch(watchId);
