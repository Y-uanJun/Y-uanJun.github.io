js笔记之记录一下String类型方法（一）

在javascript代码中，字符串是相当常见的。ECMAScript提供了多种方法以便程序员处理字符串。最近在项目中用到一些Sting类型的方法，在这里做一个小笔记。

开篇

String类型其实就是字符串的对象包装类型，可以使用new来创建。
比如 var string = new String(“hello world”);其创建的实例有一个length属性。

各类方法

字符方法charAt()和charCodeAt()


这两种方法都是以单字符字符串的形式返回给定位置的字符，不同的是charAt()方法返回的是字符，而charCodeAt()返回的是相应字符的字符编码。

从上图就可以看出，charAt(2)返回的是字符“l”，而charCodeAt()返回的是字符编码“108”。当然在javascript中要想访问字符串中特定位置的字符，可以使用方括号加数字索引的方式，比如 stringValue[2]，同样返回字符“l”。

字符操作方法
1、concat()

用于将一个或者多个字符串拼接起来。
不过，大多数时候，通常用“+”号操作符来拼接字符串

2、slice() substring() substr()

返回被操作字符串的子字符串，可以说是截取一段字符串。
这三个方法都接受两参数
第一个参数:指定字符串的开始位置。
第二个参数:对于slice()和substring()来说指定字符串的结束位置，若不填，这结束位置是字符串的最后一位。而对于substr()方法来说，则是指定返回的字符个数

当接受两个参数时，slice()和substring()返回“lo w”，从位置三开始，到位置7结束，但不包含位置7，因此不包含字符“o”；
而substr()方法则是返回“lo world”，因为第二个参数的意思是返回字符串长度为7的字符串。
同样执行者三种方法对原字符串无影响。

trim()方法

这个方法就是创建一个字符串副本，删除前置及后缀的所有空格，然后返回删除后的字符串。


从上面代码可以看出，执行了trim()方法后，原字符串前面和后面的空格都删掉了。但是trim()方法只是创建了一个字符串副本，因此原字符串不改变。

目前支持这个方法的浏览器有IE9+、Firefox3.5+、Safari5+、和Opera0.5+和Chrome。
此外，Firefox3.5+、Safari5+和Chrome 8+还支持非标准的trimLeft()——用于删除字符串开头空格；trimRight()——用于删除字符串末尾空格。

字符串分割split()

split()方法接受两个参数。
第一个参数是字符串或者是正则表达式，从该参数指定的地方分割字符串，必填
第二个参数是指定返回的数组的最大长度，可选。

split()方法实例
